# Activity2Context for OpenClaw

## Scope

This skill is an adapter layer for OpenClaw.  
It does not replace the runtime collector.

Use this skill when:
- the workspace contains `activity2context/memory.md`
- you want the agent to leverage recent behavior context
- you need a concise summary of current user intent without reading raw logs

## Expected input file

- Primary memory file: `<workspace>/activity2context/memory.md`
- Format: entity list generated by Activity2Context runtime

If the file is missing:
- state that Activity2Context memory is unavailable
- continue with normal reasoning

## Usage rules

1. Treat `activity2context/memory.md` as Active Memory.
2. Prioritize recent entities (`LastActive`) and high-duration entities.
3. Use memory as context, not as hard truth.
4. If user-provided instructions conflict with memory, follow the user.
5. Never expose sensitive raw data that is not already present in memory output.

## Suggested reasoning pattern

When relevant, extract:
- Current focus apps
- Recently edited documents
- Recently visited web references

Then infer:
- likely current goal
- probable next step
- potential ambiguity to confirm

## Operational commands (optional)

When user asks to refresh memory:
- run Activity2Context indexer command in workspace:
  - Windows: `powershell -NoProfile -ExecutionPolicy Bypass -File <workspace>/activity2context/runtime/windows/activity2contextctl.ps1 -Command index -InstallRoot <workspace>/activity2context -ConfigPath <workspace>/activity2context/config.local.json`

When user asks status:
- run Activity2Context status command:
  - Windows: `powershell -NoProfile -ExecutionPolicy Bypass -File <workspace>/activity2context/runtime/windows/activity2contextctl.ps1 -Command status -InstallRoot <workspace>/activity2context -ConfigPath <workspace>/activity2context/config.local.json`

## Safety

- Do not ask for raw behavior logs by default.
- Prefer memory entity file over raw event streams for token efficiency.
- Keep summaries compact and task-oriented.
